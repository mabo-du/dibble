Publication Figure Generator — Approaches

Three architectural approaches for generating standardised three-view lithic drawings

A / Matplotlib-based vector pipeline

Extract orthographic projections (plan/profile/section) from the mesh via vertex projection, render silhouettes and contour lines with Matplotlib, output SVG via svgutils or direct SVG generation. Scale bar and callouts as Matplotlib annotations.

Pros

  • Vector output natively (SVG, PDF)
  • Full programmatic control over every line
  • No extra rendering dependencies

Cons

  • Manual silhouette extraction from mesh
  • Harder to get publication-quality contour shading
  • Matplotlib layout can be fiddly

B / PyVista render-to-vector pipeline

Use PyVista's parallel projection to capture plan/profile/section views as images, then trace silhouettes using edge detection + vectorisation (potrace or shapely simplify). Compose into SVG layout with scale bar and callouts.

Pros

Leverages existing PyVista viewer infrastructure

Parallel projection built into VTK

Can preview figures interactively

Cons

Requires raster→vector conversion step

Edge tracing can introduce artefacts

More complex pipeline (render → trace → layout)

C / Trimesh silhouette + SVG composition (recommended)

Extract silhouettes directly from the mesh using trimesh's path module (2D cross-section projection at the mesh boundary). Render internal ridges via curvature-weighted shortest paths (inspired by Lithics3D's igraph approach). Compose everything into a standalone SVG using svgwrite, with parametric scale bars and callouts.

Pros

Pure vector from start to finish — no raster step

trimesh path module handles silhouette extraction

SVG is directly editable in Illustrator/Inkscape

Lightest dependency addition

Cons

Internal ridge rendering needs curvature algorithm work

Less visual preview during development
