Rethinking the Approach

What if we didn't have to choose between preview and vector quality?

The key insight

PyVista wraps VTK. VTK has a feature called GL2PS — it renders through OpenGL but captures vector primitives (lines, polygons, text) as SVG/PDF/EPS paths. No raster intermediate, no tracing step, no artefacts.

This means we get: preview in the viewer (same VTK pipeline) AND pure vector output in one go.

How it works

  1. User orients the mesh in the PyVista viewer (already built)
  2. Clicks Export → Publication Figure
  3. App sets camera to plan/profile/section views (parallel projection)
  4. VTK's GL2PS exports each view as a high-quality vector SVG
  5. App composes the three SVGs into a standard plate layout + scale bar + callouts + ID block
  6. Single SVG/PDF saved

Why this beats all three original options

What we gain

  • Interactive preview — see it before you export it
  • Pure vector output — no raster, no artefacts
  • Reuses existing PyVista infrastructure (no extra deps)
  • Publication-quality (VTK is used in scientific journals)
  • Zero user interaction beyond "click export"

What it costs

  • ~15 extra lines of VTK code for the GL2PS call
  • SVG composition step (three SVGs → one plate)
  • That's it — seriously
