# Dibble v3.5 — Scale Bar Auto-Detection Design

**Date:** 2026-05-27
**Status:** Approved design, pending implementation

## 1. Overview

Add automatic metric scale detection to the Dibble photogrammetry pipeline. Currently, v3 produces meshes in COLMAP's arbitrary coordinate units — shape is correct but absolute measurements (mm) are meaningless. v3.5 detects a scale reference in the photos and applies a 7-DoF similarity transformation to produce meshes in real-world units (millimetres), ready for measurement.

The existing `scale_bar_cm` field in `PhotogrammetryConfig` (unused since v3) is wired up.

## 2. Architecture

### 2.1 New module

```
lithicore/src/lithicore/
├── _scale_detection.py       # Scale detection + application
│                              # Exports: ScaleResult
│                              #          detect_scale_aruco(...) -> Optional[ScaleResult]
│                              #          detect_scale_ruler(...) -> Optional[ScaleResult]
│                              #          apply_scale_to_mesh(mesh, factor) -> Trimesh
```

### 2.2 Integration into v3 pipeline

The scale detection runs between COLMAP dense fusion and meshing:

```
COLMAP stages → fused.ply → [Scale Detection] → scaled.ply → decimation → output
                              ↑
                   photos/ + sparse cloud
```

Detection operates on the **sparse point cloud + source photos** (trusted for linear relative distances, per deep research), not the dense mesh. The scale factor is applied to the output via a simple vertex multiplication (post-hoc similarity transformation, not COLMAP bundle adjuster modification).

## 3. Three-Tier Scale Detection (Default/Guided/Expert)

Matching the v3 pipeline's existing three-tier pattern:

| Mode | Method | User effort | Accuracy |
|---|---|---|---|
| **Default** | ArUco marker detection | Print PDF, place in shot | ±0.1% |
| **Guided** | 2D ruler detection (classical CV) | Standard ruler in shot | ±1-3% |
| **Expert** | Manual scale input | Click 2 points, enter distance | ±0.5-2mm (user-dependent) |

The modes are tried in order: ArUco → ruler → manual. If the higher tier succeeds, lower tiers are skipped. If it fails, it degrades to the next.

### 3.1 Data types

```python
@dataclass
class ScaleResult:
    scale_factor: float
    method: str                    # "aruco" | "ruler" | "manual" | "none"
    confidence: float              # 0.0 – 1.0
    detected_length_mm: float      # physical length inferred (0 if manual)
    warnings: list[str]

# Unchanged config field (already exists):
# PhotogrammetryConfig.scale_bar_cm: float = 0.0
#   0.0 → auto-detect
#   >0  → known scale bar length in cm
```

## 4. ArUco Detection (Default Mode)

### 4.1 Marker specification

| Parameter | Value |
|---|---|
| ArUco dictionary | `cv2.aruco.DICT_6X6_250` |
| Marker size | 20×20 mm (default), configurable |
| Border bits | 1 |
| Print format | PDF with 10 markers per page |

### 4.2 Detection flow

```
photos/ + COLMAP cameras.bin + images.bin + points3D.bin
        ↓
   for each photo:
        cv2.aruco.detectMarkers(image, dictionary, detectorParams)
        → corners (pixel coords) + ids
        ↓
   for each unique marker id detected in ≥2 views:
        triangulate 3D corner positions using COLMAP camera poses
        (via pycolmap or direct binary parsing)
        ↓
        for each adjacent corner pair:
            compute L2 distance in arbitrary space
        ↓
        scale_factor = known_mm / mean(sparse_distances)
        ↓
   return ScaleResult with median scale across all detected markers
```

### 4.3 COLMAP data reading

COLMAP stores camera poses and point data in:
- `cameras.bin` — camera intrinsics (focal length, principal point, distortion)
- `images.bin` — image poses (quaternion + translation for each photo)
- `points3D.bin` — sparse 3D tie points

Pycolmap (`pycolmap.read_cameras_binary()`, `pycolmap.read_images_binary()`) provides direct Python access to these. This is already the most well-trodden path — tools like `aruco-estimator` use exactly this approach.

### 4.4 Fallback

If no ArUco marker is found in any photo (e.g., legacy datasets): return `None` silently, and the pipeline tries ruler detection.

## 5. Ruler Detection (Guided Mode)

### 5.1 Detection flow

```
photos/ + COLMAP sparse reconstruction
        ↓
   select 3-5 photos with best ruler visibility:
        - adaptive thresholding (cv2.adaptiveThreshold)
        - Canny edge detection + Hough Line Transform
        → locate ruler longitudinal edges + orientation
        ↓
   extract pixel intensity profile along ruler axis
        → find tick mark positions via
          scipy.signal.argrelextrema (local minima)
        ↓
   compute median tick spacing in pixels
        ↓
   cross-validate across selected photos
        → reject outliers >2σ from median
        → mean pixel-per-mm ratio
        ↓
   scale_factor = mean(pixel_per_mm) × depth_scaling
```

### 5.2 Assumptions

- Scale bar has high-contrast alternating marks (standard black-on-white ruler)
- Scale bar is approximately planar with the turntable surface
- At least 3 photos show the ruler clearly with minimal blur
- Optimal photos are those where the ruler is most fronto-parallel
  (detected by longest continuous Hough line segment)

### 5.3 Limitations (documented)

- Glossy/reflective rulers may fail (specular highlights violate Lambertian assumption)
- Very short rulers (<20 mm visible) may not have enough tick marks for reliable spacing
- Textured backgrounds may cause false positives
- Accuracy degrades with extreme perspective foreshortening

## 6. Manual Scale Input (Expert Mode)

### 6.1 Flow

If both automatic methods fail:
1. Pipeline completes with unscaled mesh (COLMAP arbitrary units)
2. Result page shows a warning: "Scale not detected."
3. User clicks "Set Scale Manually" button
4. User clicks two points on the mesh in the 3D viewer
5. A dialog prompts: "Enter distance between these points (mm):"
6. User types the known distance
7. Mesh rescales immediately

### 6.2 Implementation

The 3D viewer (`_viewer_3d.py`) already supports click-to-pick points (used for landmarks and manual orientation). Reusing that interaction:
- Enter "scale set" mode
- Two clicks → two 3D coordinates
- Distance = sqrt((x1-x2)² + (y1-y2)² + (z1-z2)²)
- scale_factor = entered_mm / calculated_distance
- Multiply all vertices by scale_factor

## 7. Integration into v3 Pipeline

### 7.1 Modified run_pipeline() flow

```python
# After dense MVS (fused.ply created), before meshing:
scale_result = None

if config.scale_bar_cm == 0:
    # Auto-detect: try ArUco first, fall back to ruler
    if colmap_available():
        scale_result = detect_scale_aruco(
            photos, sparse_cloud, marker_size_mm=20.0
        )
    if scale_result is None:
        scale_result = detect_scale_ruler(photos, sparse_cloud)
else:
    # User specified known scale bar length — try ArUco or skip to manual
    scale_result = detect_scale_aruco(
        photos, sparse_cloud, marker_size_mm=config.scale_bar_cm * 10
    )

if scale_result is not None:
    # Apply scale before meshing
    scaled_ply = workspace / "scaled.ply"
    mesh = trimesh.load(str(fused_ply))
    mesh.vertices *= scale_result.scale_factor
    mesh.export(str(scaled_ply))
    # Use scaled_ply for meshing instead of fused_ply
```

### 7.2 If scale detection fails entirely

The pipeline completes with a warning. The result page shows "Scale not detected — measurements will be in arbitrary units" and offers the manual input button.
