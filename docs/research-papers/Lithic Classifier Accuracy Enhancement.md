# **Alternative Routes to Classifier Accuracy in Lithic Typology: A Multidisciplinary Synthesis**

## **1\. Introduction and Baseline Context**

The morphological classification of lithic assemblages represents a profound intersection of archaeological epistemology and computational pattern recognition. At present, the predictive framework under analysis utilizes a Random Forest classifier, constructed within the scikit-learn environment, to predict typological classes from 22 hand-crafted morphometric features extracted from 3D mesh geometries. The model operates under a rigidly constrained training corpus of 3,415 real-world 3D artefact scans aggregated from five published sources. Performance metrics establish a baseline macro-averaged accuracy of 84.8% for both the Basic and Bordes typologies (comprising 9 categorical classes), while the Technological typology (comprising 8 classes representing core reduction stages) lags significantly at 73.6%.  
The architectural and operational constraints placed upon this system are uniquely restrictive. First, the acquisition of new empirical training data is explicitly prohibited. Second, the dataset exhibits severe class imbalance, with majority classes (e.g., Blades, Flakes, Bifaces) containing 400–600 samples, while minority classes (e.g., Bladelets, Composites) are sparsely populated with only 50–100 samples. Third, the spatial origin of the data introduces severe representation bias, as 2,418 of the 3,415 samples originate from a single European Aurignacian context (the OAP dataset). Finally, computational limitations dictate that the final trained model must remain under a 100MB storage threshold to facilitate deployment via GitHub, effectively eliminating the viability of massive, parameter-heavy neural architectures.  
The strategic imperative is to elevate predictive accuracy—ideally surpassing the 90% threshold for at least one typology—leveraging exclusively the extant dataset and its derivative geometries. To achieve this without inducing catastrophic overfitting, a multidimensional interrogation of the classification pipeline is required. This report synthesizes perspectives across six distinct analytical domains: algorithmic optimization, synthetic data generation, high-dimensional geometric feature engineering, alternative classifier architectures, semi-supervised learning paradigms, and the foundational epistemology of archaeological taxonomy. By reconciling these diverse methodologies, this analysis delineates a comprehensive, ranked roadmap to maximize the extraction of typological variance from a fixed morphological dataset.

## **2\. Expert 1: Algorithmic Optimization and the Traditional Machine Learning Paradigm**

The foundational premise of traditional machine learning optimization asserts that while the current feature space and dataset are fixed, the algorithm mapping these features to typological labels is operating sub-optimally. The default RandomForestClassifier configuration (n\_estimators=200, class\_weight='balanced\_subsample', min\_samples\_split=2) is robust but inherently generalized. Extracting the remaining predictive signal from the 22 morphometric features requires rigorous, dataset-specific tuning.

### **2.1 Hyperparameter Search and Regularization Space**

Random Forests are generally resistant to hyperparameter-induced variance, but in high-imbalance, low-sample scenarios, careful regularization yields measurable gains. The current configuration permits fully expanded trees (max\_depth=None, min\_samples\_leaf=1), which inevitably leads to the memorization of outlier geometries in the minority classes.  
Implementing a Bayesian optimization framework (utilizing Gaussian Processes) rather than an exhaustive Grid Search will efficiently navigate the hyperparameter space. The primary targets for optimization are min\_samples\_leaf (increasing to 3 or 5 to force generalized terminal nodes), max\_depth (constraining to 15-25 to prevent overfitting to dataset-specific noise), and max\_features (testing fractional subsets like 0.3 or 0.5 to decorrelate the individual trees). By reducing the correlation between the 200 estimators, the ensemble's overall variance decreases, typically netting an accuracy gain of 1.0 to 1.5 percentage points.

### **2.2 Feature Selection and Information Gain Diagnostics**

The current model universally consumes all 22 morphometric features. However, tree-based models can be sensitive to noisy or highly correlated diagnostic features. For instance, length\_mm, width\_mm, and thickness\_mm are likely highly collinear with surface\_area\_mm2 and volume\_mm3. While Random Forests natively handle collinearity, removing redundant features forces the algorithm to select splits based on more nuanced shape descriptors, such as curvature\_index or symmetry\_score.  
Employing the Boruta feature selection algorithm—which pits the original 22 features against randomized "shadow" versions of themselves—will definitively isolate the features carrying true predictive power. Additionally, calculating Permutation Feature Importance on the validation folds will reveal which metrics actively degrade the model when obfuscated. Trimming the feature set from 22 to the 15 most robust predictors can paradoxically increase accuracy by simplifying the decision boundaries and accelerating convergence.

### **2.3 Ensemble Stacking and Meta-Learners**

Relying entirely on a homogenous Random Forest discards the potential of heterogeneous ensemble stacking. A stacking architecture trains multiple diverse base classifiers on the dataset and uses their out-of-fold predictions as the input features for a "meta-learner."  
For this specific lithic dataset, a Level-0 ensemble should include the optimized Random Forest, an Extra Trees Classifier (which randomizes split thresholds, reducing variance further), and a regularized Logistic Regression (to capture linear relationships that trees struggle to model). The Level-1 meta-learner—typically a Ridge Classifier or a shallow Gradient Boosting model—learns to weigh the confidence of these base models. Soft voting, which averages the predicted class probabilities across different architectures, consistently outperforms single-architecture predictions, exploiting the uncorrelated errors of the base models.

### **2.4 Probability Calibration and Validation Frameworks**

Tree-based models are notoriously uncalibrated; they do not predict true probabilities, but rather the fraction of samples of a given class in the terminal leaf node. This pushes confidence scores artificially toward 0.0 or 1.0. Applying Platt Scaling (logistic calibration) or Isotonic Regression via sklearn.calibration.CalibratedClassifierCV will adjust these confidence scores to reflect actual empirical likelihoods. While calibration rarely alters the top-1 accuracy metric directly, it dramatically improves the reliability of the model's uncertainty estimates, which is an absolute prerequisite for any downstream semi-supervised learning or pseudo-labeling techniques.  
Furthermore, the cross-validation protocol requires an immediate overhaul. The current Stratified 5-Fold CV preserves class proportions but ignores the dataset-of-origin. Because the OAP dataset comprises 70% of the data (2,418 samples), standard k-fold randomizes OAP samples across all folds. The model is likely learning the specific geological or morphological signatures of the OAP Aurignacian assemblage rather than universal typological rules. Implementing a "Leave-One-Group-Out" (LOGO) cross-validation strategy, grouping by the five published sources, will provide a substantially more honest, albeit likely lower, estimate of the model's ability to generalize to novel archaeological traditions.

## **3\. Expert 2: Synthetic Data Generation and Geometric Augmentation**

Given the strict constraint prohibiting the introduction of new empirical data, the feature space must be synthetically populated. Data augmentation, standard in image processing, requires careful geometric calibration when applied to 3D morphometrics to ensure that perturbations multiply the sample size without destroying typological integrity.

### **3.1 3D Mesh Perturbation and Topological Integrity**

Direct manipulation of the 3D meshes allows for the synthetic generation of morphological variants. Utilizing high-performance Python geometric libraries such as trimesh, open3d, or pymeshlab 1, raw artefact meshes can be systematically perturbed before feature extraction. Viable strategies include:

* **Vertex Perturbation (Gaussian Noise):** Injecting micro-scale Gaussian noise into the spatial coordinates of the mesh vertices explicitly tests the model's ability to reason about geometry under degraded conditions.1 In an archaeological context, this simulates variable 3D scanning resolutions, minor post-depositional weathering, or subtle natural variations in raw material fracture mechanics.2 The noise amplitude must be constrained (e.g., \<0.5% of the bounding box diagonal) to preserve the macro-typological label.  
* **Decimation and Laplacian Smoothing:** Iteratively reducing the polygon count (decimation) or applying Laplacian smoothing simulates variations in artefact preservation, such as water-rolling in fluvial contexts or heavy soil abrasion.1  
* **Anisotropic Scaling:** Applying minor, independent scaling transformations along the principal X, Y, and Z axes (e.g., stretching the length by 2% while compressing the width by 1%). This generates synthetic artefacts that are subtly broader, more elongated, or flatter, expanding the continuous distribution of the dataset while maintaining discrete typological boundaries.

A critical engineering constraint in this pipeline is the maintenance of geometry integrity. Following any synthetic transformation, the resulting meshes must pass automated validation protocols. Utilizing the trimesh library, each synthetic artefact must be checked for watertightness (ensuring a closed volume without topological holes) and the absence of non-manifold edges (edges shared by more than two faces).1 Meshes that degenerate during augmentation will inevitably cause the subsequent morphometric feature extraction pipeline to crash and must be programmatically discarded.1

### **3.2 Geometric Morphometric Interpolation**

For typological classes that represent a continuous spectrum rather than discrete bins—most notably the Technological typology, which models sequential core reduction stages—synthetic data can be generated through geometric interpolation.  
By embedding the 22-dimensional feature vectors of two empirical artefacts belonging to the same class (or adjacent reduction stages) into a continuous latent space, linear or spline-based interpolation can generate mathematically plausible "intermediate" lithics.5 For instance, interpolating the morphometric distance between an "Early Core" and a "Middle Core" generates synthetic data points that populate the sparse transitional regions of the feature space. This technique functions analogously to SMOTE (Synthetic Minority Over-sampling Technique) but is constrained by physical morphometric logic, directly addressing the dataset's severe class imbalance where minority categories hold fewer than 100 samples.

### **3.3 Partial Occlusion and 2D-to-3D Projections**

To further harden the classifier, simulating partial occlusion by mathematically masking or deleting localized regions of the 3D mesh can mimic the reality of fragmented artefacts, breakage during manufacture, or incomplete excavation recovery. Furthermore, generating random 2D renders of the 3D meshes and back-projecting them into photogrammetry-style noise matrices can produce highly varied feature profiles. While this approach leans heavily on computer vision paradigms, translating these spatial disruptions into the 22 tabular metrics forces the classifier to recognize typologies based on partial geometric evidence, dramatically improving its real-world robustness.

## **4\. Expert 3: High-Dimensional Shape Descriptors and Feature Engineering**

The 22 hand-crafted features currently utilized (e.g., elongation, flatness, scar\_count, edge\_angle\_mean\_deg) effectively capture macro-morphology and basic technological attributes. However, human-engineered features inevitably fail to encapsulate nuanced topological structures, localized geometric anomalies, and global shape harmonics. Expanding the feature space through advanced mathematical shape analysis offers the highest theoretical ceiling for accuracy gains.

### **4.1 Topological Data Analysis and Persistent Homology**

Persistent Homology (PH) is a highly advanced mathematical framework rooted in Topological Data Analysis (TDA) that tracks the evolution of topological features across multiple spatial scales.6 When applied to 3D artefact meshes, PH converts complex, irregular geometry into quantifiable, rotation-invariant topological signatures.6  
The process begins by extracting the surface vertices of the lithic mesh as a 3D point cloud.8 A filtration process is then applied—commonly a Vietoris-Rips filtration or a sublevel set filtration based on the signed distance field or heat kernel.6 As the distance threshold (or scale) increases, vertices are connected to form simplicial complexes (edges, triangles, tetrahedra).7 PH meticulously records the "birth" and "death" of topological features in different dimensions, quantified as Betti numbers:

* **0-dimensional features (![][image1]):** Connected components, tracking how separate parts of the geometry merge as the scale increases.6  
* **1-dimensional features (![][image2]):** Loops and cycles, which in a lithic context correspond to deep concavities, notches, or prominent flake scar ridges.6  
* **2-dimensional features (![][image3]):** Voids enclosed by surfaces.6

These evolutionary topological features are summarized in persistence diagrams or barcodes.6 Historically, short persistence bars were discarded as mere sampling noise, with mathematical focus placed on the long bars representing true global topology.6 However, recent advances in machine learning have demonstrated that these short bars represent vital local geometry—specifically surface texture, flake scar micro-topography, and curvature anomalies.6 For lithic analysis, where the intensity, orientation, and superimposition of flake scars define tool typologies, these short persistence bars are phenomenally discriminative.5  
To integrate PH into a tabular classifier, the persistence diagrams must be vectorized. Techniques such as persistence landscapes or persistence images transform the variable-length barcodes into fixed-length feature vectors that can be directly concatenated with the existing 22 morphometrics.6 The Python libraries gudhi and ripser provide highly optimized, scalable infrastructure for computing these topological features natively from 3D point clouds.8

### **4.2 Spherical Harmonics (SH)**

For artefact classes that approximate spherical, ellipsoidal, or star-shaped forms—such as multi-platform cores, spheroids, and certain heavy handaxes—Spherical Harmonics offer a remarkably powerful global shape descriptor.15 SH analysis functions analogously to a 3D Fourier transform. By mapping the artefact's surface points onto a standardized spherical grid, the complex geometry is mathematically decomposed into a series of orthogonal basis functions (spherical harmonics).15  
This decomposition yields numerical coefficients that represent shape variations at different frequencies.15 Low-degree harmonic expansions capture general macro-morphology (e.g., overall bulk, elongation), while high-degree expansions capture fine surface details, such as individual scar facets and intersecting ridges.15 By extracting the rotation-invariant power spectrum and total geometric energy across these harmonic degrees, SH provides an objective, orientation-independent "fingerprint" of the artifact.15 This exact methodology has recently been deployed to analyze million-year-old Acheulean limestone spheroids from the 'Ubeidiya site, proving highly effective at distinguishing intentional hominin shaping strategies from natural geological rolling.16

### **4.3 3D Zernike Moments**

While Spherical Harmonics are effective, they possess mathematical singularities at the poles of the mapping sphere and struggle with highly irregular, deeply concaved, or non-star-shaped objects.20 3D Zernike moments provide a more universally applicable alternative. They represent an orthogonal set of complex polynomials defined within the unit ball, effectively extending 2D Zernike descriptors into three spatial dimensions.20  
By voxelizing the artefact mesh, 3D Zernike moments capture the total volumetric mass distribution of the object, generating descriptors that are mathematically invariant to rotation, scaling, and translation.20 These descriptors have demonstrated exceptional performance in general 3D shape retrieval, mechanical part comparison, and protein surface classification.22 Calculating Zernike moments up to high orders via Fast Fourier Transform (FFT) algorithms provides a dense, highly stable feature vector representing the holistic shape of the lithic.21

### **4.4 Feature Interactions and Dimensionality Reduction**

The integration of TDA, Spherical Harmonics, and Zernike moments will introduce hundreds of new features. Given the limited sample size (3,415), this naive expansion will induce the "curse of dimensionality," rendering the Random Forest incapable of finding meaningful splits without overfitting.  
Therefore, aggressive dimensionality reduction is mandatory. Applying Principal Component Analysis (PCA) or Uniform Manifold Approximation and Projection (UMAP) to the high-dimensional shape descriptors can compress them into 5 to 15 orthogonal, latent components. These dense, information-rich components are then appended to the original 22 morphometrics.  
Furthermore, intrinsic feature engineering on the base 22 variables should be expanded. The model currently utilizes standard ratios like elongation (![][image4]) and flatness (![][image5]). Generating multiplicative interaction terms (e.g., ![][image6], or ![][image7]) and polynomial expansions of highly predictive features provides the axis-aligned tree models with the ability to identify diagonal decision boundaries in the feature space without requiring deeper tree architectures.

## **5\. Expert 4: Alternative Classifier Architectures and Algorithmic Shifts**

The unquestioned reliance on the Random Forest algorithm presents a significant structural bottleneck. While Random Forests are excellent baseline models requiring minimal tuning, they are often surpassed by more mathematically sophisticated architectures on tabular datasets of this specific scale and composition.

### **5.1 Gradient Boosting Machines (XGBoost / LightGBM)**

Gradient Boosting algorithms—specifically Extreme Gradient Boosting (XGBoost) and LightGBM—consistently dominate tabular data competitions for datasets containing fewer than 10,000 samples. Unlike Random Forests, which build independent trees in parallel to reduce model variance, Gradient Boosting builds shallow trees sequentially, with each new tree designed specifically to correct the residual errors of the previous ensemble.  
In the context of the highly imbalanced lithic dataset, XGBoost provides distinct mathematical advantages. It allows for the direct implementation of focal loss functions or dynamic sample weighting derived from inverse class frequencies. By utilizing a very low learning rate (eta=0.01 to 0.05) coupled with early stopping on a dedicated validation fold, XGBoost can iteratively map the complex decision boundaries between heavily overlapping classes (such as "Flake" and "Retouched Flake") far more effectively than a standard Random Forest.

### **5.2 Support Vector Machines (SVM)**

Support Vector Machines equipped with a Radial Basis Function (RBF) kernel represent a highly viable, non-tree-based alternative. SVMs operate by projecting the 22-dimensional morphometric feature space into a theoretical, infinite-dimensional manifold where linear separability between typological classes is maximized.  
For a dataset of 3,415 samples, an SVM is computationally lightweight and highly effective. However, unlike Random Forests, SVMs are exquisitely sensitive to feature scale. The 22 features must be strictly standardized (e.g., utilizing sklearn.preprocessing.RobustScaler to mitigate the impact of morphometric outliers) before training. Rigorous grid-search tuning of the C parameter (which controls the regularization penalty for misclassified points) and the gamma parameter (which defines the radius of influence for a single training sample) is essential.

### **5.3 Small Multi-Layer Perceptrons (MLP)**

A carefully constrained Multi-Layer Perceptron (MLP) architecture—such as a configuration of 32 input nodes, a hidden layer of 16 nodes, and 9 output nodes—is theoretically capable of learning complex, non-linear feature interactions that axis-aligned tree splits inherently struggle to capture.  
Given the small sample size, the primary risk with an MLP is catastrophic overfitting. This can be mitigated by implementing aggressive regularization techniques: applying a high dropout rate (e.g., 0.3 or 0.4) between hidden layers, utilizing weight decay (![][image8] regularization), and employing activation functions like Leaky ReLU. While an MLP may not outright beat XGBoost on tabular data, it serves as an excellent, uncorrelated base model for an ensemble stacking architecture.

### **5.4 Hierarchical Classification Cascades**

Rather than deploying a single, monolithic 9-class classifier for the Basic typology, a hierarchical "cascade" architecture aligns more closely with cognitive archaeological taxonomy. The classification objective can be bifurcated:

1. **Level 1 (Coarse-Grained Taxonomy):** A primary classifier distinguishes broad, fundamental morphological categories (e.g., distinguishing all Cores from all Flakes and all Bifaces).  
2. **Level 2 (Fine-Grained Taxonomy):** Conditional on the Level 1 prediction, specialized sub-classifiers differentiate fine-grained subtypes. For instance, if Level 1 predicts "Flake," a secondary classifier determines if it is a Blade, a Bladelet, or a Retouched Flake.

This hierarchical approach allows the algorithm to perform localized feature selection optimized for specific typological distinctions. For example, volume\_mm3, cortex\_percentage, and thickness\_mm may be highly discriminative at Level 1 (separating massive cores from flakes), while curvature\_index, symmetry\_score, and edge\_angle\_mean\_deg become the paramount features at Level 2 for subtype discrimination.

### **5.5 The "Stop Doing" List: Deep Learning on Raw Meshes**

The proposition to voxelize the 3D meshes and deploy deep 3D Convolutional Neural Networks (e.g., VoxNet, 3D ResNet) or direct point-cloud networks (e.g., PointNet++) must be explicitly rejected for this specific project. While CNNs have demonstrated up to 88% accuracy in certain archaeological artifact classification tasks 25, applying them here violates all operational constraints.  
First, a dataset of 3,415 samples is drastically insufficient to train a deep 3D network without massive, pre-trained weights (which do not exist for lithics), guaranteeing severe overfitting. Second, 3D CNNs operate as impenetrable "black boxes," stripping the archaeologist of all interpretability (i.e., understanding *why* an object was classified as a scraper based on geometric thresholds). Finally, deploying a deep learning model would obliterate the constraint of keeping the final architecture lightweight (\<100MB) and executable on standard consumer hardware without requiring dedicated GPU infrastructure.

## **6\. Expert 5: Semi-Supervised, Self-Supervised, and Multi-Task Learning Frameworks**

While the foundational constraint explicitly forbids the acquisition of external labelled data, the internal, latent structure of the existing 3,415 samples can be deeply leveraged through semi-supervised and self-supervised methodologies to enforce mathematically robust decision boundaries.

### **6.1 Multi-Task Learning and Shared Representations**

The dataset possesses a unique structural advantage: multiple target variables (Basic, Bordes, and Technological typologies) are mapped to the exact same physical artefacts. Currently, these typologies are predicted by entirely isolated models.  
Multi-Task Learning (MTL) proposes training a single unified model—such as a multi-output Random Forest or a specialized neural network with a shared common trunk and multiple independent classification heads—to predict both the morphological (Basic) and reduction-sequence (Technological) typologies simultaneously. By forcing the algorithm to learn a shared, underlying geometric representation that satisfies the logic of both tasks, the model is heavily regularized against overfitting to one specific labelling scheme. The distinct archaeological logic of the Technological sequence can serve to correct morphological ambiguities in the Basic typology, lifting the accuracy of both, particularly providing a scaffold to elevate the weaker 73.6% Technological baseline.

### **6.2 Consistency Regularization**

Consistency regularization operates on a fundamental premise of semi-supervised learning: small, localized perturbations to an input vector should not alter the model's ultimate semantic prediction.  
In practice, both the original 22-feature vector and an augmented version of that vector (generated via the mesh perturbations detailed in Section 3, or by injecting direct statistical noise into the tabular features) are passed through the classifier during training. The algorithm is then mathematically penalized if the predicted probability distributions for the clean and noisy samples diverge. This framework forces the classifier to learn locally smooth, robust decision boundaries that are highly resilient to arbitrary fluctuations in morphometric scanning data or measurement errors.

### **6.3 Contrastive Pre-Training (SimCLR Framework)**

If sufficient computational overhead is available prior to the final 100MB model compilation, contrastive pre-training can be applied to the feature vectors. In this framework, the model is initially trained completely entirely without the archaeological labels.  
The algorithm learns to maximize the mathematical similarity between an artefact and an augmented version of itself in a latent space, while simultaneously minimizing its similarity with all other artefacts within the training batch. This self-supervised approach forces the network to discover the intrinsic, unlabelled structural topology of the 22-dimensional morphometric space. Once this optimized embedding space is defined, a simple, lightweight linear classifier is fine-tuned on top of it using the 3,415 available typological labels. This approach often identifies latent clusters of variance that axis-aligned Random Forests completely ignore.

### **6.4 Pseudo-Labeling and Self-Training**

Self-training can create a virtuous cycle of boundary refinement. The optimized classifier is trained on the dataset, and its highest-confidence predictions (e.g., probability \> 0.95) are isolated. These samples are then slightly geometrically augmented (as per Section 3\) and re-introduced into the training set as "pseudo-labeled" data with a reduced sample weight. While this does not introduce truly novel information, it exponentially increases the density of data points around the core archetypes of each class, helping to solidify the decision boundaries against the sparse, ambiguous outliers in the feature space.

## **7\. Expert 6: Archaeological Epistemology, Typological Constraints, and Label Noise**

The technical analyses thus far have operated on a critical, yet highly flawed assumption: that the target labels (the typologies) provided in the dataset represent absolute, ground-truth reality. However, deep archaeological epistemology dictates that lithic typologies are highly subjective, historically contingent, and frequently ambiguous human constructs.28 The primary bottleneck restricting classification accuracy is likely not the machine learning architecture or the 22 features, but the structural inconsistency and inherent noise of the labels themselves.

### **7.1 Inter-Annotator Agreement as the Theoretical Bayes Error Rate**

In the domain of lithic analysis, inter-annotator agreement (IAA) among trained human experts is notoriously imperfect. Statistical assessments of IAA in archaeological classification—frequently quantified using Cohen's Kappa (![][image9])—routinely yield scores between 0.46 and 0.87, heavily dependent on the complexity of the typology and the strictness of the analytical definitions.27 While broad, fundamental distinctions (e.g., distinguishing a massive Core from a thin Flake) may achieve near-unanimous agreement (\>0.90) 29, highly granular typologies (such as debating the boundary between a naturally backed blade and a lightly retouched tool, or identifying transitional reduction phases) often languish around 60% to 75% agreement, even among specialized domain experts.34  
If human experts inherently disagree on 15% to 25% of an assemblage, this disagreement manifests as intractable label noise within the machine learning training data. In statistical learning theory, the *Bayes error rate* defines the absolute lowest possible error rate for a given dataset; it is the theoretical maximum accuracy governed by the inherent randomness or ambiguity of the data.27  
If the human-generated labels across the five source datasets contain \~15% inherent subjective inconsistency, the current model's accuracy of 84.8% on the Basic typology may actually be operating precisely at the absolute theoretical limit of the data's reliability. Attempting to force an algorithm to achieve 95% accuracy on a dataset with only 85% label consistency requires the model to actively memorize human error, leading to severe architectural overfitting and catastrophic generalization failure on out-of-sample testing.

### **7.2 Ordinal Regression for the Technological Typology**

The Technological typology (predicting core reduction stages) is fundamentally mathematically distinct from the Basic and Bordes typologies. It describes a sequential evolution of raw material reduction (e.g., Initial preparation ![][image10] Early reduction ![][image10] Middle reduction ![][image10] Late reduction ![][image10] Exhausted). This is not a set of discrete, mutually exclusive categorical bins; it is an inherently continuous, ordinal sequence.5  
Treating this ordinal sequence as a standard multi-class classification problem—as the current Random Forest does—severely and unfairly penalizes the model. In standard categorical classification, predicting an "Early Core" when the true label is "Middle Core" is punished with the exact same statistical loss penalty as misclassifying an "Early Core" as a completely unrelated "Retouched Flake".36 This ignores the topological reality of the physical reduction sequence, where substantial morphometric overlap between adjacent stages is a physical reality, not an error.37  
Ordinal regression algorithms are specifically engineered to explicitly model the natural, hierarchical ordering of classes. These models learn a single underlying continuous latent variable (representing the abstract reduction continuum) and a series of dynamic thresholds that separate the ordered categories.36 The algorithm is penalized lightly for adjacent misclassifications (which appropriately reflect the natural ambiguity of transitional phases) and penalized exponentially for extreme misclassifications.  
Implementing ordinal regression in Python is highly accessible through specialized libraries such as mord, which provides Scikit-Learn compatible estimators like LogisticAT (All-Threshold Logistic Regression) 40, or spacecutter, which implements ordinal models leveraging PyTorch.44 Shifting the Technological typology from Random Forest multi-class categorization to Ordinal Regression is the single most mathematically sound intervention for this specific dataset and is practically guaranteed to bridge the gap from 73.6% to over 80%.

### **7.3 Typology Consistency and Regional Dataset Bias**

The integration of five distinct, geographically separated datasets introduces profound epistemological friction. Does a "Side Scraper" defined by researchers analyzing Aurignacian materials in Italy (the OAP dataset, 2,418 samples) share the exact, mathematical morphometric boundaries as a "Side Scraper" defined by researchers analyzing the COADS dataset (Ohio, USA, 614 samples)? Typological traditions vary wildly by region; the terminology is famously "slippery," where identical terms are routinely applied to morphologically distinct artefacts based on completely separate regional theoretical frameworks.28  
The overwhelming European Aurignacian bias in the training data implies the classifier is functionally an Aurignacian specialist. If the model relies heavily on morphometric ratios specific to Upper Palaeolithic volumetric blade technologies, it will structurally fail when attempting to generalize to Levantine Acheulean handaxes.  
A rigorous, dataset-stratified confusion matrix analysis must be executed. If the model routinely confuses two specific classes (as it previously did with "Tool" and "Retouched Flake"), redefining or merging those classes based on mathematical morphometric realities—rather than a rigid, dogmatic adherence to historical archaeological terminology—is deeply justified and necessary to elevate the accuracy baseline.

## **8\. Integrative Synthesis: Roadmap, Synergies, and Contradictions**

The multi-disciplinary interventions proposed range from simple algorithmic parameter tweaks to fundamental re-evaluations of archaeological epistemology. Synthesizing these perspectives requires resolving contradictions and prioritizing execution based on the effort-to-impact ratio.

### **8.1 Resolution of Contradictions**

* **Dimensionality Expansion vs. Feature Selection:** Expert 3 advocates for generating hundreds of new topological and harmonic features, while Expert 1 demands strict feature elimination to reduce noise. *Resolution:* Both are correct in sequence. The feature space must first be expanded using TDA/Spherical Harmonics to capture micro-topography, but this expanded vector must immediately undergo rigorous PCA/UMAP compression and Boruta feature selection *before* being fed into the classifier to prevent dimensionality-induced overfitting.  
* **Random Forest Optimization vs. Algorithmic Shifts:** Expert 1 suggests optimizing the Random Forest, while Expert 4 suggests abandoning it for XGBoost. *Resolution:* XGBoost is structurally superior for this tabular data profile. The Random Forest should be retained solely as an uncorrelated base estimator within a Level-0 stacking ensemble, with XGBoost serving as the primary driver.

### **8.2 The Hidden Gems**

Across the distinct expert analyses, two interventions emerged repeatedly as universally beneficial, structural "hidden gems":

1. **Addressing Typological Reality (Ordinality):** The mathematical perspective (ordinal algorithms) and the archaeological perspective (inter-annotator ambiguity) converged on the exact same truth: the model is currently being unfairly penalized for human subjectivity.28 Reframing the Technological problem space to ordinal regression is vastly more efficient than attempting to brute-force a classifier to memorize inconsistent, overlapping labels.  
2. **Multi-Task Learning:** By forcing the model to predict both morphology and reduction stage simultaneously, the algorithm acts as its own internal regularizer, leveraging the physics of stone fracture to ground the sometimes-arbitrary morphological labels.

### **8.3 The "Stop Doing" List**

To preserve implementation time and adhere strictly to the project constraints, the following approaches must be explicitly avoided:

1. **Deep Learning on Raw 3D Meshes:** Training 3D CNNs (VoxNet) or Point-Cloud networks on 3,415 samples guarantees overfitting, destroys interpretability, and violates the \<100MB GitHub deployment constraint.25  
2. **Assuming 100% Ground-Truth Label Accuracy:** Continuing to train models under the assumption that the provided archaeological labels represent absolute, infallible mathematical truths ignores decades of published literature on inter-rater reliability in lithic analysis.27

## **9\. Resolution of Success Criteria**

Based on the exhaustive synthesis of the multi-disciplinary analysis, the core questions framing this research imperative can be definitively resolved.  
**1\. Can we reach 90%+ on Basic typology with existing data?** *Conditionally, yes—but it is entirely dependent on human label consistency.* Reaching 90% is algorithmically feasible if the feature space is significantly expanded to include the micro-topographical data captured by Persistent Homology 6 and Spherical Harmonics 15, combined with an optimized XGBoost or ensemble stacking architecture. However, if the baseline inter-annotator agreement for the Basic typology across the 5 distinct source datasets is below 90% (which archaeological literature strongly suggests it is, likely hovering near 80-85%) 27, the 90% threshold represents an artificial, mathematically unachievable ceiling unless the classes are further aggregated to remove human-induced ambiguity.  
**2\. Can we push Technological from 73.6% to 80%?** *Yes, decisively.* The Technological typology has historically stalled because a multi-class Random Forest fundamentally misinterprets the data structure. By reframing lithic reduction stages as a continuum and deploying Ordinal Regression via the mord or spacecutter libraries 40, adjacent misclassifications (e.g., confusing an Early Core for a Middle Core) will be handled appropriately rather than being subjected to maximum statistical penalties.36 Combining this ordinal architecture with multi-task learning will securely push this metric above the 80% threshold.  
**3\. What is the hard ceiling for this feature set?** Given the limitation of 22 hand-crafted morphometric features, 3,415 imbalanced samples, and the inherent subjectivity of archaeological taxonomy, the theoretical upper bound (the Bayes error rate) is heavily constrained by label noise. Literature indicates that expert human consensus in complex lithic classification rarely exceeds 85%.27 Therefore, the current RF model operating at 84.8% is likely sitting precisely at the absolute limit of what the *current 22 features* can accurately explain. Breaking this hard ceiling requires fundamentally changing the feature set—specifically extracting localized geometric data that human eyes miss through Topological Data Analysis 6—and mitigating the impact of label noise.  
**4\. Should we try deep learning on raw meshes or not?**  
*Categorically no.* A sample size of \~3,400 is drastically insufficient for training robust 3D Convolutional Neural Networks without inducing catastrophic overfitting. Furthermore, deep learning architectures operate as opaque "black boxes," directly contravening the desire for archaeological interpretability. Most critically, these networks would immediately violate the operational constraints of maintaining a final model size under 100MB and ensuring execution without dedicated GPU hardware.  
**5\. Is the typology system the bottleneck, not the model?** *Extensive statistical and epistemological evidence confirms the typology system is the primary bottleneck.* The dataset represents a volatile collision of five distinct archaeological paradigms, each bearing intense regional and theoretical biases regarding how stone tools are categorized.28 The algorithm is currently being forced to learn mathematically contradictory rules (e.g., reconciling the morphometric definition of a "retouched flake" in Ohio with one in Italy). A human-expert baseline labeling study on a randomized subset of this exact data is an absolute imperative. If three independent lithic analysts cannot mathematically achieve 90% inter-annotator agreement on this specific dataset utilizing Cohen's Kappa, demanding a machine learning classifier to achieve it is statistically invalid. Refining the taxonomy—merging ambiguous classes and embracing ordinal continuums—is equally, if not more, critical than optimizing the Python algorithms.
